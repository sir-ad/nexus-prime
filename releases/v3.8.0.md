# Nexus Prime v3.8.0

## Summary
Nexus Prime now behaves like an orchestrator-first agent operating system instead of a runtime that callers have to manually wire together. This release promotes the autonomy orchestrator into the control plane, adds raw-prompt execution and artifact discovery surfaces, persists orchestration/token/client truth per runtime, and updates the shared dashboard to reflect that persisted truth instead of transient host-process state.

## Highlights
- New `nexus_orchestrate` raw-prompt MCP entrypoint for bounded autonomous execution
- New discovery MCP APIs for skills, workflows, hooks, and automations
- Orchestrator-first routing for high-level Nexus execution and dashboard-triggered runs
- Persisted orchestration session snapshots, token summaries, token timelines, and per-run token drilldowns
- Dashboard token analyzer plus lifetime compression metrics that survive restart
- Primary-client precedence that correctly prefers active Codex evidence over stale Claude footprints
- Rewritten `AGENTS.md` protocol with context-acquisition order, subsystem trigger rules, and worker handoff guidance
