# Nexus Prime v3.12.0

## Summary
`v3.12.0` is a public-surface release that makes Nexus Prime easier to evaluate, easier to trust, and easier to ship. The README, website, changelog, and generated inventory now tell one coherent story around bootstrap, orchestrate, worktree-backed execution, memory fabric, session-first RAG, token budgeting, and runtime truth.

## What Changed
- Rebuilt the README hierarchy around product definition, install, bootstrap-orchestrate flow, proof screens, capability families, runtime contract, generated registry, and generated runtime catalog
- Replaced repeated website imagery with a graph-first cockpit hero, a dedicated runtime-sequence proof module, and a distinct knowledge-trace view
- Added stronger public capability framing for orchestration, worktree swarms, memory fabric, RAG gating, token budgeting, runtime truth, bootstrap surfaces, and release governance
- Converted the changelog to version-level `<details>` sections so GitHub history is easier to scan without losing depth
- Expanded generated inventory summaries so README, docs, and dashboard all report the same counts and surface areas

## Public-Surface Improvements
- Stable screenshot aliases now back the main public proof surfaces instead of version-stamped file names
- Screenshot containers render at natural responsive sizes instead of looking shrunken inside oversized frames
- Landing-page proof modules now separate cockpit view, runtime sequence, and knowledge trace instead of repeating one overview image
- README now includes release history and deeper capability framing without disturbing the existing badge and widget block
- Docs tests and public-surface tests now enforce the new screenshot references and release-surface structure

## Runtime and Reliability Improvements Already Carried In This Branch
- Worktree doctor now heals stale Nexus-owned git worktree metadata before runtime and verifier worktree creation
- Runtime truth now records worktree health and surfaces shorter remediation-oriented MCP failures instead of raw git walls
- Generated feature registry, catalog pages, and dashboard inventory all reflect the same shared source of truth
- Release QA remains gated behind build, lint, tests, packaging smoke, production audit, and release smoke
