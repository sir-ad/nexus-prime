# Nexus Prime v3.2.2

## Summary
This release hardens the restored topology-first dashboard so it stays usable when older MCP/dashboard processes are still running locally. The dashboard now detects incompatible listeners, starts on the next free port when needed, hydrates partial data instead of blanking the page, and renders a cleaner lower-density UI.

## Highlights
- Added a dashboard compatibility contract on `/api/health` with:
  - `dashboardApiVersion`
  - capability flags for required routes
  - active dashboard URL
  - dashboard mode (`bound` or `reused`)
- Replaced silent occupied-port bridging with:
  - compatibility probe on the default dashboard port
  - compatible-listener reuse
  - automatic fallback to the next free local port for incompatible listeners
- Reworked dashboard hydration to:
  - use per-resource status tracking
  - render partial data when only some routes fail
  - show compatibility/degraded banners instead of an empty shell
- Normalized legacy event payloads client-side so the live stream stays readable.
- Reduced typography and padding across the rails and event stream to make the dashboard less congested.

## Validation
- `npm run build`
- `npm test`
- `npm run lint`

## Notes
- `npm run lint` remains green with existing legacy warnings in older untouched files outside this dashboard recovery scope.
- If an older MCP/dashboard process is still holding `3377`, use the newer dashboard URL printed by the latest process after startup.
