# Nexus Prime v3.9.0

## Summary
Nexus Prime now compiles one shared machine-facing instruction packet for orchestrated work instead of relying on raw `AGENTS.md`, duplicated prompt text, or client-specific adapter stubs. This release adds the cross-client instruction gateway, persists execution ledgers and compiled packets in the runtime/dashboard surfaces, and tightens the split between the human operator manual and the model-facing runtime handoff.

## Highlights
- New shared `InstructionGateway` that compiles `AGENTS.md`, `.agent/rules/*`, and runtime-selected context into a token-budgeted packet
- Cross-client envelope renderers for Codex, Claude Code, Antigravity/OpenClaw, Cursor, Windsurf, and Opencode
- Persisted execution ledgers plus new dashboard APIs: `/api/orchestration/ledger` and `/api/instruction-packet`
- Compact human-facing `AGENTS.md` with machine-facing packet handoff under `.agent/runtime/packet.json` and `.agent/runtime/packet.md`
- Canonical runtime snapshot behavior that preserves the parent run's orchestration truth when bounded continuation children execute
