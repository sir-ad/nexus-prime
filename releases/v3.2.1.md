# Nexus Prime v3.2.1

## Summary
This release restores the dashboard to the earlier topology-first concept while keeping the new runtime real. The dashboard now exposes memory topology, POD activity, client presence, skills, workflows, a live normalized stream, and a guarded local control plane backed by real runtime APIs.

## Highlights
- Restored the topology-first dashboard layout with:
  - connected ecosystem rail
  - center topology canvas for memory, runs, and POD
  - live stream rail
  - persistent inspector drawer
- Added dashboard APIs for:
  - memory snapshots and memory network inspection
  - POD worker and signal views
  - client registry visibility
  - normalized event feed
- Added a real `ClientRegistry` with heartbeat-first and heuristic-second detection for:
  - Codex
  - Claude Code
  - Antigravity
  - Opencode
  - MCP
- Added safe local control-plane actions to:
  - execute runtime tasks
  - deploy/revoke skills
  - deploy/run workflows
  - reconnect/clear client entries
- Added persisted-run fallback so recent runs remain visible after refresh.

## Validation
- `npm run build`
- `npm test`
