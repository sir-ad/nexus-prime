# Nexus Prime v3.6.0

## Summary
Nexus Prime now ships a native specialist-orchestrator overlay on top of the existing real runtime. This release adds a first-party specialist roster, preset crews, visible planner state across MCP and dashboard surfaces, and a non-regressing planning path that can fall back to the stable baseline runtime when confidence is low.

## Highlights
- Generated native specialist roster with preset crew templates for product, engineering, GTM, content, finance, security, and research work
- Additive task planner that selects crew, specialists, skills, workflows, tools, swarm mode, fallback plan, review gates, and continuation output
- New MCP planning and roster APIs: `nexus_plan_execution`, `nexus_list_specialists`, and `nexus_list_crews`
- Dashboard planner surfaces for specialist browser, crew browser, planner preview, and live planning ledger
- Runtime planner state persisted into run artifacts and emitted as `planner.stage` events
- Tool policy now enforced per worker role, with explicit action-driven runs kept backward-compatible

## Validation
- `npm run build`
- `npm test`
- `npm run lint`

## Notes
- `npm run lint` still reports the repo’s long-standing warning set in older untouched files outside this release surface.
- The previous baseline runtime path remains available via `NEXUS_SPECIALIST_PLANNER_DISABLED=1`.
