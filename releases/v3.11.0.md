# Nexus Prime v3.11.0

## Summary
Nexus Prime now ships a stronger orchestration-time context layer instead of relying mostly on repo files plus memory recall. This release adds the new Knowledge Fabric, session-first RAG collections, a pattern registry, by-source token telemetry, and a hardened release path that keeps local and GitHub Actions quality gates aligned.

## Highlights
- Knowledge Fabric execution bundles now combine repo context, memory, attached RAG collections, reusable patterns, and prior runtime traces
- Session-first RAG collections can be created, ingested, attached, retrieved, and inspected through the orchestrator, MCP, and dashboard surfaces
- New pattern registry supports reusable orchestration and retrieval recipes without embedding external app code into the runtime
- `CI & Publish` now enforces build, lint, full tests, and `npm pack --dry-run` on pull requests before any publish flow
- RAG collection storage is hardened against path traversal and remote URL ingestion now times out cleanly on hanging hosts
- TypeScript tests and public-surface checks now execute through `tsx`, keeping local and GitHub Actions behavior in sync

## Validation
- `npm run build`
- `npm run lint`
- `npm test`
- `npm pack --dry-run`

## Notes
- Release tag: `v3.11.0`
- Untracked local `.playwright-cli/` and `.taskmaster/` artifacts remain outside this release scope and were not included.
